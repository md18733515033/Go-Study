简单:25个关键字
高效:编译的强类型语言,垃圾回收,可以使用指针直接进行内存访问
生产力:语法简洁,支持复合

一:go run hello.go和go build hello.go两种执行流程方式的区别
    1. 如果我们先编译生成了可执行文件,那么我们可以将该可执行文件拷贝到没有go开发环境的机器上,仍然可以运行
    2. 如果我们是直接go run hello.go,那么如果要在另外一个机器上运行,也需要go开发环境,否则无法执行
    3. 在编译时,编译器会将程序运行依赖的库文件包含在可执行文件中,所以,可执行文件变大了很多
二:编译和运行的说明
    1.有了go源文件,可以通过编译器将其编译成机器可识别的二进制文件
    2.在该源文件目录下,通过go build -o name hello.go 文件进行编译,生成制定的name可执行文件(name处可自定义),在windows中必须是.exe后缀
    3.如果程序没有错误,没有任何提示,会在当前目录下出现一个可执行文件,该文件时二进制编码文件,也可以是可执行的程序
    4.如果程序有错误,编译时,会在错误的地方报错,有助于排错.例如:"./hello.go:10:2: undefined: fmt.Printnl",表示第10行第二个单词,没有定义
    5.运行有第一条两种形式
三:应用程序入口
    1.必须是main包
    2.必须是main()方法,func main()
    3.包名不必须是main,但是程序内的packege main是必须的
四:传参
    1.main函数不支持传入参数  func main(arg []string)  错误
    2.在程序中直接通过os.Args获取命令行参数
